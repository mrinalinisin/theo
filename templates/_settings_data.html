Migration bundle — import
Upload a theo_export.zip produced by any Theo instance. Products are matched by URL — existing ones are left untouched, new ones are added. Tags, currencies, and brands are reused if already present, otherwise created. Image files are renamed on the way in so they never collide with what you already have.
Nightly backups
Theo writes a full migration bundle to instance/backups/ automatically every night at midnight. If the machine was asleep at midnight, the missed backup runs the first time you open the app the next day. The newest {{ backup_retention }} backups are kept; older ones are pruned.

Danger zone — reset app
Wipes everything from this Theo instance: every product, tag, brand, purchase, See Also link, published-page record, and every image file in instance/images/.
Preserved: Settings (so your GitHub token survives) and Currency reference rows (so the New-listing form still offers $/₹/€). If you want to keep the data, download the migration bundle at the top first — reset cannot be undone.
