How to generate a token
Quick path — Classic PAT (simpler):
  1. Open github.com → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token.
  2. Pick a Note (e.g. "Theo publish") and an expiry.
  3. Tick public_repo if your target repo is public, or the full repo scope if it's private.
  4. Click Generate token, copy it, paste above.
Scoped path — Fine-grained PAT (more secure, more clicks):
  1. Open github.com → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token.
  2. Set a Token name and expiry.
  3. Under Repository access, choose Only select repositories and pick your target (e.g. username.github.io).
  4. Scroll down to Permissions → Repository permissions, click + Add permissions.
  5. In the picker, tick the checkbox next to "Contents" (the picker is a checkbox list, not a row of dropdowns). Close the picker.
  6. A Contents row now appears in the Permissions block with Access: Read. Click "Read" and change it to "Read and write".
  7. Click Generate token, copy it, paste above.
