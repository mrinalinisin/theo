# Theo

> A personal inventory app for things you buy.

Theo keeps track of products you care about — what you're watching, what you've ordered, and what's arrived — with images, tags, prices, expected delivery dates, and notes. It's a **catalogue**, not a scraper or a price tracker.

The catalogue is fed by **Roger**, a Safari extension that clips products from any web page into Theo with one click. Roger does the page-reading; Theo does the storage, organisation, and reporting. If you want price alerts, point an external tool like OpenClaw at the URLs Theo stores.

---

## Features

- **Inventory by status** — watching → purchased → shipped → received, each with its own view
- **Tags** — coloured, freeform; created on the fly from the browser extension
- **Cart & checkout** — bulk-mark a set of watched items as purchased in one go
- **Reports** — spending by tag (pie chart), monthly summaries grouped by tag or merchant
- **Publish** — push a curated subset of items as a static HTML page to your own GitHub Pages site (one-click)
- **Stories** (optional) — generate J. Peterman-style copy for each item via the Anthropic Claude API
- **PWA** — installable on iOS (Add to Home Screen) and macOS (Add to Dock)
- **Responsive** — desktop, tablet, phone

---

## Requirements

- **Python 3.10+**
- A modern browser. Safari if you want to use the Roger extension; any browser works for direct use.

That's the whole list. Theo uses SQLite (no separate DB process) and runs on Flask's built-in dev server by default.

## Quickstart

```bash
# 1. Clone
git clone https://github.com/mrinalinisin/theo.git
cd theo

# 2. Virtual environment
python3 -m venv venv
source venv/bin/activate

# 3. Install deps
pip install -r requirements.txt

# 4. Configure
cp .env.example .env
# Edit .env — at minimum, set SECRET_KEY to a random string.
# Generate one with:
python3 -c 'import secrets; print(secrets.token_urlsafe(48))'

# 5. Run
python3 app.py
```

Open <http://localhost:5000> (or whatever `PORT` you set).

The first run creates `instance/theo.db` (SQLite) and `instance/images/` automatically. There's no migration step.

## Configuration

All configuration lives in `.env` (see `.env.example` for the full list with comments).

| Variable | Required | Purpose |
|----------|----------|---------|
| `SECRET_KEY` | **Yes** | Flask session signing key. Replace the default before any real use. |
| `ANTHROPIC_API_KEY` | No | Unlocks the Stories feature. Get a key at [console.anthropic.com](https://console.anthropic.com). The app runs fine without it. |
| `PORT` | No | Server bind port. Defaults to `5000`. |
| `THEO_PROJECT_URL` | No | URL shown in the "Generated by Theo" footer of published pages. Defaults to the upstream repo; fork operators may override. |

GitHub publishing (used by the **Publish** feature) is configured **inside the app**, not via env vars — go to Settings → GitHub and paste a personal access token with `repo` scope.

## Adding products

The primary ingestion path is the **Roger** Safari extension, which POSTs JSON to `/products/new_from_browser`. The endpoint is unauthenticated by design (it runs on your own machine).

If you don't have Roger installed, you can call the endpoint directly:

```bash
curl -X POST http://localhost:5000/products/new_from_browser \
  -H "Content-Type: application/json" \
  -d '{
    "url":   "https://example.com/item",
    "name":  "Thing",
    "price": 1500,
    "currency": "INR",
    "tags":  ["Clothes"]
  }'
```

The endpoint canonicalises the URL, resolves/creates tags, saves images to disk, and computes perceptual hashes for duplicate detection. See [docs/architecture.md](docs/architecture.md#how-does-product-ingestion-work) for the full ingestion flow.

## Running in the background

`python3 app.py` is fine for daily local use, but if you want Theo to survive logouts and restart on crash, run it under a process supervisor (runit, systemd, launchd, supervisord — pick your platform). The maintainer's setup uses runit; the run script and lifecycle commands are documented in [docs/architecture.md](docs/architecture.md#how-is-the-theo-process-managed).

For LAN/Tailscale access (and HTTPS, which iOS PWA install requires), see [docs/architecture.md](docs/architecture.md#how-do-i-reach-theo-from-another-device).

## Project layout

```

├── models.py               # SQLAlchemy models: Product, Purchase, Tag, …
├── config.py               # Env-var-driven configuration
├── image_store.py          # On-disk image persistence + perceptual hashing

├── static/                 # CSS, service worker, icons, manifest
```

The `instance/` directory (created at first run) holds the SQLite DB and image files. It's gitignored.

## Tech stack

- **Flask** 3.x + **Flask-SQLAlchemy** on **SQLite**
- **Pillow** for image processing and perceptual hashing
- **requests** for fetching remote images during Roger ingestion
- **python-dotenv** for `.env` loading
- **anthropic** SDK for optional Story generation

No scrapers, no background workers, no third-party messaging. The server is single-process Flask; one writer at a time at the SQLite layer is the bottleneck and that's fine for personal use.
